# ユーザーマニュアル

本ドキュメントは `feelfuse` の基本的な使い方を説明します

---

## 概要

本機は Wi-Fi または Ethernet でネットワークに接続し、UDP で受信した OSC メッセージを処理します\
主な用途は、以下の通りです

- 触覚データの振動子への出力
- サーボ制御

設定の変更には SD カードを使用します\
ファームウェア更新は、SD カード、OSC コマンド、またはブラウザページから実行できます

fermware v0.2.0 では、以下の構成が有効です

| 項目 | 値 |
| ------ | ---- |
| ネットワーク | Wi-Fi / Ethernet |
| 既定のネットワークモード | Wi-Fi AP |
| 既定の SSID | `FTW_DEVICE` |
| 既定の Wi-Fi パスワード | `FTW_PSWD` |
| 既定の IP アドレス | `192.168.4.1` |
| mDNS ホスト名 | `ftw-device` |
| UDP ポート | `3333` |
| SD カード | 有効 |
| OTA 更新 | 有効 |

---

## 通信の仕様

### 接続先

本機は UDP ソケットを作成し、指定されたポートで OSC メッセージを待ち受けます

| 項目 | 値 |
| ------ | ---- |
| プロトコル | UDP |
| メッセージ形式 | OSC |
| 既定ポート | `3333` |
| 既定 mDNS 名 | `ftw-device.local` |

ポート番号は SD カードの `network.ini` にある `[udp] port` で変更できます\
`network.ini` がない場合は、既定値の `3333` が使われます

### 通信コマンド一覧

| OSC アドレス | ペイロード | 説明 |
| -------------- | ------------ | ------ |
| `/tactile` | blob 1 個 | 触覚出力データを I2S に送信します |
| `/force` | blob 1 個 | サーボ制御データを UART に転送します |
| `/ota/update` | なし | SD カード上の `firmware.bin` からファームウェア更新を開始します |

OSC bundle は未対応です通常の OSC message として送信してください

### ペイロードの説明

#### `/tactile`

`/tactile` は blob を 1 個受け取ります\
blob サイズは 1024 bytes です

| 項目 | 値 |
| ------ | ---- |
| OSC 型 | `b` |
| blob 数 | 1 |
| サイズ | 1024 bytes |
| スロット数 | 4 |
| フレーム数 | 256 |

受信した blob データは以下の様に処理され、4 スロット分の TDM バッファとして I2S に書き込まれます

- 1024 bytes のデータは 256 byte ずつに分割され、各スロットに割り当てられる
- 各 byte 中央の値（7F）を基準として波形を出力

サイズが 1024 bytes でない場合、そのメッセージは処理されません

本機では、8khzのサンプリング周期のデータとして再生を行います\
送信するデータは8khzのサンプリングのものを使用するか、送信時に8khzとしてリサンプリングしてください

パケット1個で、32ms分のデータとなります\
おおむね32ms間隔でパケットを送信してください

デバイスは最大で8パケット分のバッファを持ちます\
送信開始時に、数パケット分のデータをまとめて送信することで、のちのパケットの遅延による再生の断絶を回避することができます\
最低でも開始時に2パケット分のデータを送信することを推奨します

リアルタイム送信などの場合、頭に無音のパケットを送信することで同様のバッファリングの効果を受けることができます\
ただし、バッファリングの分だけ再生が遅延することになりますので、環境に合わせて最適な値を選択してください

#### `/force`

`/force` は blob を 1 個受け取ります\
受信した blob は UART にそのまま送信されます

サーボ制御データのワイヤ形式は、接続先サーボコントローラの仕様に合わせて送信してください\
コマンドの送信周期は、`/tactile` コマンドに合わせて、32ms周期程度を想定しています

#### `/ota/update`

`/ota/update` はペイロードを使用しません\
コマンドを受信すると、SD カードのルートにある `firmware.bin` を読み込んで OTA 更新を開始します

SD カードの初期化に失敗した場合、または `firmware.bin` が見つからない場合は更新に失敗し、電源 LED が赤く点滅します

## ボタンの仕様

操作ボタンが1つあります

| 操作 | 動作 |
| ------ | ------ |
| 短押し | 現状、機能は割り当てられていません |
| 長押し 5 秒以上 | 保存済みネットワーク設定をクリアし、再起動します |

短押しは、ボタンを押して離したタイミングで実行されます\
長押しは、押下状態が 5 秒以上続いたタイミングで実行されます

## LED の仕様

LED が2つあります

- 電源 LED
- ネットワーク LED

| LED | 条件 | 色 / 表示 |
| ----- | ------ | ----------- |
| 電源 LED | 起動中 | 青 |
| 電源 LED | 初期化完了 | 緑 |
| 電源 LED | OTA 開始または実行中 | 白系 |
| 電源 LED | OTA 成功 | 緑点滅 |
| 電源 LED | OTA 失敗 | 赤点滅 |
| ネットワーク LED | 接続中 | 青 |
| ネットワーク LED | 接続完了 | 緑 |
| ネットワーク LED | Wi-Fi エラー | 赤 |
| ネットワーク LED | UDP パケット受信 | 一時的に白 |

UDP 受信がない状態が続くと、受信通知用の白表示は解除されます

## SD カードの仕様

SD カードは FAT32 形式を推奨します\
起動時に SD カードがない場合でも、本機は SD カード機能なしで起動を継続します

### `network.ini`

ネットワーク設定を変更する場合は、SD カードのルートに `network.ini` を配置します

```ini
; FEELTECHWear Network Configuration
; このファイルを SD カードのルートに "network.ini" として配置してください
; 記載しない項目は sdkconfig のデフォルト値が使われます

[mode]
type = wifi       ; wifi または ethernet
role = ap         ; ap または sta

[wifi_ap]
ssid     = FTW_DEVICE
password = FTW_PSWD
channel  = 1
max_sta  = 4

[wifi_sta]
ssid     = Target SSID
password = Target Password

[network_ap]
ipv4_addr    = 192.168.4.1
ipv4_mask    = 255.255.255.0
ipv4_gateway = 192.168.4.1

[mdns]
hostname = ftw-device

[udp]
port = 3333

[browser_update]
enabled  = false      ; true または false
password = ftw-update
```

| セクション | 項目 | 説明 |
| ------------ | ------ | ------ |
| `[mode]` | `type` | `wifi` または `ethernet` を指定します（`eth` も Ethernet として扱われます） |
| `[mode]` | `role` | `ap` または `sta` を指定します |
| `[wifi_ap]` | `ssid` | AP モード時の SSID です |
| `[wifi_ap]` | `password` | AP モード時のパスワードです |
| `[wifi_ap]` | `channel` | AP モード時の Wi-Fi チャンネルです |
| `[wifi_ap]` | `max_sta` | AP に同時接続できる最大台数です |
| `[wifi_sta]` | `ssid` | STA モード時の接続先 SSID です |
| `[wifi_sta]` | `password` | STA モード時の接続先パスワードです |
| `[network_ap]` | `ipv4_addr` | AP モード時の IP アドレスです |
| `[network_ap]` | `ipv4_mask` | AP モード時のサブネットマスクです |
| `[network_ap]` | `ipv4_gateway` | AP モード時のゲートウェイです |
| `[mdns]` | `hostname` | mDNS ホスト名です |
| `[udp]` | `port` | UDP 待ち受けポートです |
| `[browser_update]` | `enabled` | ブラウザページからのファームウェア更新を有効にします。既定値は `false` です |
| `[browser_update]` | `password` | ブラウザページからの更新に使用するパスワードです。既定値は `ftw-update` です |

`network.ini` が読み込まれると、ネットワーク設定は NVS に保存されます\
一度保存されたネットワーク設定は、次回以降 SD カードがなくても使用されます\
`[browser_update]` の設定は NVS には保存されず、SD カードの `network.ini` からのみ読み込まれます\
保存済み設定を消したい場合は、操作ボタンを 5 秒以上長押しして再起動してください

## ファームウェアアップデート手順

本機は SD カードの `firmware.bin`、またはブラウザから選択したファームウェアファイルを使ってファームウェアを更新できます\
SD カード上の `firmware.bin` を使う場合は、OSC コマンドまたはブラウザページから更新を開始できます\
ブラウザからファイルをアップロードして更新する場合は、SD カードへ保存せず、そのまま OTA 更新に使用します

ブラウザ経由でのアップデート機能を使用する場合は、SD カードの `network.ini` で `[browser_update] enabled` を `true` にする必要があります\
安全のため、この設定は本体に保存されません\
ブラウザからアップデートを行う場合は、設定ファイルを配置したSDカードをセットしてください

### ファームウェアを準備する

弊社指定の場所から、`firmware.bin` をダウンロードしてください\
Github: <https://github.com/commissure-inc/FeelFuse>

### SDカードからアップデートする手順

SD カードから更新する場合は、SD カードのルートに `firmware.bin` を配置します

#### 1. SD カードにコピーする

SD カードのルートに `firmware.bin` をコピーします

```text
SDカード/
└── firmware.bin
```

#### 2. SD カードを挿入して起動する

`firmware.bin` を配置した SD カードを本機に挿入し、本機を起動します\
SD カードが正しく初期化されると、OTA 更新の準備ができます

#### 3.A. OSC経由で実行

本機の UDP ポートに OSC メッセージ `/ota/update` を送信します\
ペイロードは不要です

#### 3.B. ブラウザ経由で実行

ブラウザで本機のステータスページを開きます\
例: <http://ftw-device.local/>

`Firmware Update` 欄の `Password` に更新用パスワードを入力し、`Update from SD card` を押します\
初期設定の更新用パスワードは `ftw-update` です

### アップロードからアップデートする手順

アップロードから更新する場合は、ブラウザで選択した `.bin` ファイルを直接 OTA 更新に使用します

#### 1.ブラウザで実行

ブラウザで本機のステータスページを開きます\
例: <http://ftw-device.local/>

`Firmware Update` 欄の `Password` に更新用パスワードを入力し、`.bin` ファイルを選択して `Update from upload` を押します\
初期設定の更新用パスワードは `ftw-update` です

### アップデート実行中

更新が開始されると、電源 LED が白系の表示になります\
更新に成功すると緑点滅後に再起動します\
失敗した場合は赤点滅します

### 更新後の起動を確認する

自動ロールバックが有効です\
更新後の初回起動で初期化が完了すると、新しいファームウェアは有効として確定されます\
起動に失敗した場合は、前のファームウェアへ戻ります

## Config内容の確認

ブラウザからhttpでデバイスにアクセスすると、現在のConfigの内容を確認することができます\
一部の値（パスワードなど）は、セキュリティのため固定長でマスク表示され、実際の文字数を示しません\
例: <http://ftw-device.local/>

同じ内容は JSON API からも確認できます\
例: <http://ftw-device.local/api/status>

ブラウザからの更新操作は、内部的には以下のAPIを使用します

| API | 内容 |
| ------ | ---- |
| `POST /api/update/sd` | SD カード上の `firmware.bin` から更新します |
| `POST /api/update/upload` | リクエスト本文の `.bin` ファイルから更新します |

どちらの更新APIも、`X-FTW-Update-Password` ヘッダに更新用パスワードを指定する必要があります

## トラブルシューティング

### 通信できない

- UDP ポートが `3333`、または `network.ini` の `[udp] port` と一致しているか確認してください
- Wi-Fi AP モードの場合は、接続先 SSID が `FTW_DEVICE` になっているか確認してください
- `network.ini` で `type = ethernet` を指定している場合は、Ethernet ケーブルとネットワーク機器を確認してください
- OSC bundle ではなく、通常の OSC message を送信してください

### `/tactile` が反応しない

- OSC アドレスが `/tactile` になっているか確認してください
- ペイロードが blob 1 個になっているか確認してください
- blob サイズが 1024 bytes になっているか確認してください

### `/force` が反応しない

- OSC アドレスが `/force` になっているか確認してください
- ペイロードが blob 1 個になっているか確認してください
- ペイロードの内容が、指定のサーボのコマンド形式に合致しているか確認してください

### `network.ini` が反映されない

- SD カードのルートに `network.ini` があるか確認してください
- ファイル名が小文字の `network.ini` になっているか確認してください
- SD カードが FAT32 形式であるか確認してください
- 保存済み設定を消したい場合は、操作ボタンを 5 秒以上長押しして再起動してください

### OTA 更新に失敗する

#### 共通

- 電源 LED が赤点滅している場合は、SD カード未初期化、ファイルなし、読み込み失敗、書き込み失敗などが考えられます

#### SDカード経由

- SD カードのルートに `firmware.bin` があるか確認してください
- ファイル名が小文字の `firmware.bin` になっているか確認してください
- SD カードが正しく挿入されているか確認してください

#### ブラウザ経由

- ブラウザページに更新操作が表示されない場合は、SD カードの `network.ini` で `[browser_update] enabled` が `true` になっているか確認してください
- ブラウザ更新の場合は、更新用パスワードが正しいか確認してください
- ブラウザからの SD カード更新で初期化に失敗する場合は、`SDカード経由` の内容も併せて確認してください
- ブラウザからアップロードする場合は、正しい `.bin` ファイルを選択しているか確認してください

### 起動ループする

OTA 更新後に新しいファームウェアが正常起動できない場合、自動ロールバックにより前のファームウェアへ戻ります\
しばらく待ってから再度起動状態を確認してください
